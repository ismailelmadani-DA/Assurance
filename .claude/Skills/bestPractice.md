This skill enforces the D&A Technologies "Normes de Developpement Salesforce" covering:
- Naming conventions for objects, fields, record types, layouts, validation rules, lightning pages
- Layered architecture pattern (DM > EM > SM > TH/BAT/LWC)
- Apex class naming (DMXXX, EMXXX, SMXXX, THXXX, BATXXX, SCHXXX, WSXXX, WRXXX, LCXXX, LWCXXX, VFXXX)
- Code conventions (no hardcoded IDs, no SOQL in loops, bulk-safe methods, documentation cartouches)
- Test class standards (90% coverage target, naming, seeAllData=true forbidden)
- Configuration norms (picklists, custom fields, flows)
